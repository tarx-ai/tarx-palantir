# tarx-palantir

**Umbrella beachhead** for TARX × Palantir-class enterprise environments.

Local-first **sovereign AI runtime** (TARX-OS) as the open, zero-lock-in layer under Foundry / AIP / Apollo-class lifecycle thinking.

## Status

- Exploration + open connector beachhead  
- Production connector code: [`tarx-palantir-connector`](https://github.com/tarx-ai/tarx-palantir-connector)  
- This repo: docs, specs, and AI-builder onboarding for the Palantir-adjacent wedge  

## What we claim

- TARX-OS = local-first sovereign runtime + **bridge infrastructure**  
- Open Enterprise/Gov weights path; human/local weights closed & user-controlled  
- OpenAI-compatible local hop so AIP Logic can call on-device inference  
- **Complementary** to closed enterprise stacks — not a Foundry/Apollo clone  

## What we do *not* claim

- Official Palantir partnership or endorsement  
- Live Apollo product integration (conceptual lifecycle language only today)  
- Exposure of private Supercomputer routing  

## Quick start (builders)

```bash
# Prefer the connector for runnable examples
git clone https://github.com/tarx-ai/tarx-palantir-connector
cd tarx-palantir-connector
npm install
# Point AIP / OpenAI client at http://127.0.0.1:11435/v1
```
